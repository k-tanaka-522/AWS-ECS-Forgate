# AWS インフラ構成（業務管理システム）

## 1. システム概要
本システムは、業務管理を目的としたWebアプリケーションです。AWSを活用し、スケーラブルかつ安全な環境を提供します。

## 2. システム要件
本システムのインフラ設計における要件は以下の通りです。

### **耐障害性**
- RDSをマルチAZ構成とし、障害発生時に自動フェイルオーバーを実施
- ECSのタスクを複数AZに分散配置し、冗長性を確保
- ALBを利用したヘルスチェックと自動フェイルオーバーの設定
- S3データの複数リージョンへのレプリケーション（必要に応じて）

### **性能要件**
- ECSのオートスケール機能により負荷分散を実施
- RDSのリードレプリカを活用し、読み取り負荷を分散
- CloudFrontを利用し、キャッシュ管理によるレスポンスの高速化
- X-Rayによるパフォーマンス監視とボトルネック分析

### **セキュリティ要件**
- Cognitoによる認証管理を実装
- IAMロールの最小権限設計（Least Privilege）を徹底
- ALB + WAF によるWebセキュリティ強化
- CloudTrail による監査ログの取得と管理
- S3のデータをサーバーサイド暗号化（SSE-S3またはSSE-KMS）

## 3. 構築担当範囲
本プロジェクトにおけるAWSインフラ構築の担当範囲は以下の通りです。
- **VPCおよびサブネットの設計・構築**（開発・ステージング・本番の3環境）
- **ECS（Fargate または EC2）によるアプリケーション実行基盤の構築**
- **ALBの設計・構築およびSSL/TLS設定**
- **Amazon RDSの設計・構築（Auroraも検討）**
- **Amazon S3を活用したバックアップおよびログ管理**
- **CloudWatch、X-Ray、CloudTrail を用いた監視・監査基盤の構築**
- **CI/CD（CodeDeploy）を利用したデプロイ環境の構築**

## 4. 構築環境
- 本システムは以下の3つの環境で構築される
  - **開発環境（dev）**
  - **ステージング環境（stg）**
  - **本番環境（prd）**
- 各環境はそれぞれ別のAWSアカウントで管理する
- サンプル構築では開発環境（dev）を利用して検証を行う

## 5. システム構成

### **ネットワーク**
- VPC（/16 CIDR, サンプル環境では /16 を使用）
- **サブネット**：各環境ごとに適切にサブネットを割り当て
- **パブリックサブネット**：
  - ALB（Application Load Balancer）
  - NAT Gateway（プライベートサブネット用）
- **プライベートサブネット**：
  - ECS（Fargate または EC2）
  - RDS（マルチAZ構成）

### **コンピュート**
- **ECS on Fargate**
  - オートスケール（タスク数自動調整）
  - ECRにDockerイメージを管理
  - CodeDeployを活用したBlue/Greenデプロイ
- **ALB**
  - SSL/TLS対応（HTTPS）
  - Cognito認証連携

### **データベース**
- **Amazon RDS for MySQL（Auroraも選択肢）**
  - マルチAZ構成
  - 自動バックアップ（7日間保持）
  - IAM認証 + Secrets Managerで管理

### **ストレージ**
- **S3**
  - バックアップ用（DBスナップショット、ログ）
  - CloudWatch Logsのローテーション用
  - ライフサイクル管理（1年後アーカイブ、3年後削除）

### **監視・ログ管理**
- **CloudWatch Logs**
  - 必要なログのみ送信し、コストを最適化
  - 2週間保持後、S3へアーカイブ
- **X-Ray**
  - ECSタスクごとのパフォーマンス監視
- **CloudTrail**
  - 監査ログの取得
- **SNS通知**
  - 障害発生時に管理者へ通知
- **Bedrock AI による障害分析**
  - CloudWatch LogsやX-Rayのデータを分析し、異常を自動検出
  - 障害の予兆を解析し、SNS通知でアラートを送信
  - 分析結果を基に、運用チームへ推奨アクションを提示
  - **一次調査の自動化**：Bedrock AIが障害発生時のログやメトリクスを解析し、影響範囲や発生原因を特定
  - **対応の優先度付け**：重大度に応じた対応優先度を判断し、対応手順を提示

## 6. 環境テスト（WordPress導入）

### **手順**
1. **ECS（Fargate）でコンテナ起動**
   - WordPressコンテナをデプロイ
   - RDS（MySQL）接続
   - ALB経由で公開
2. **S3 + CloudFront で静的コンテンツ配信**
3. **CloudWatch Logs でログ確認**
4. **ALB + HTTPS 設定**
