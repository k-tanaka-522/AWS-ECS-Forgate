# ECSデプロイ手順書

## 1. デプロイの概要

### 1.1 デプロイ対象
- Webアプリケーションコンテナ
- ミドルウェア設定ファイル
- 環境変数

### 1.2 デプロイ環境
- 開発環境（dev）
- ステージング環境（stg）
- 本番環境（prd）

## 2. Webコンテンツのデプロイ手順

### 2.1 コンテナイメージのビルドとプッシュ
```bash
# イメージのビルド
docker build -t escforgate-app:latest ./ECS/app

# ECRへのプッシュ
aws ecr get-login-password --region ap-northeast-1 | docker login --username AWS --password-stdin ${AWS_ACCOUNT_ID}.dkr.ecr.ap-northeast-1.amazonaws.com
docker tag escforgate-app:latest ${AWS_ACCOUNT_ID}.dkr.ecr.ap-northeast-1.amazonaws.com/escforgate-app:latest
docker push ${AWS_ACCOUNT_ID}.dkr.ecr.ap-northeast-1.amazonaws.com/escforgate-app:latest
```

### 2.2 ECSタスク定義の更新
```bash
# CloudFormationの場合
aws cloudformation deploy \
  --template-file lac/cloud\ formation/compute.yaml \
  --stack-name ${ENV}-escforgate-compute \
  --parameter-overrides \
    Environment=${ENV} \
    ContainerImage=${AWS_ACCOUNT_ID}.dkr.ecr.ap-northeast-1.amazonaws.com/escforgate-app:latest

# Terraformの場合
cd lac/terraform
terraform workspace select ${ENV}
terraform apply -var="container_image=${AWS_ACCOUNT_ID}.dkr.ecr.ap-northeast-1.amazonaws.com/escforgate-app:latest"
```

### 2.3 ECSサービスの更新
- 新しいタスク定義が適用されると、ECSサービスは自動的に新しいタスクにローリングアップデートを実行
- デプロイの進行状況はAWSコンソールまたは以下のコマンドで確認可能：
```bash
aws ecs describe-services \
  --cluster ${ENV}-escforgate-cluster \
  --services ${ENV}-escforgate-service
```

## 3. 設定ファイルの管理

### 3.1 設定ファイルの管理方針
- 環境変数ファイル（.env）は各環境のタスク定義で管理
- 機密情報はAWS Secrets Managerで管理
- アプリケーション設定はコンテナイメージにバンドル

### 3.2 設定ファイルの更新手順
1. ローカルで設定ファイルを更新
2. 新しいコンテナイメージをビルド
3. ECSタスクの再デプロイ

## 4. 環境変数の管理

### 4.1 環境変数の種類
- アプリケーション設定
- データベース接続情報
- API認証情報

### 4.2 環境変数の管理方法
1. 非機密情報
   - ECSタスク定義のenvironmentセクションで定義
   - 環境ごとにCloudFormation/Terraformのパラメータで管理

2. 機密情報
   - AWS Secrets Managerで管理
   - ECSタスク定義のsecretsセクションで参照

### 4.3 Secrets Managerの設定手順
1. シークレットの作成
```bash
# シークレットの作成（例：データベースパスワード）
aws secretsmanager create-secret \
  --name ${ENV}/wordpress/db-password \
  --description "WordPress database password" \
  --secret-string "your-secure-password"

# シークレットの作成（例：データベースホスト）
aws secretsmanager create-secret \
  --name ${ENV}/wordpress/db-host \
  --description "WordPress database host" \
  --secret-string "your-db-endpoint"
```

2. シークレットの更新
```bash
# シークレット値の更新
aws secretsmanager update-secret \
  --secret-id ${ENV}/wordpress/db-password \
  --secret-string "new-secure-password"
```

3. シークレットの一覧表示
```bash
# 特定のプレフィックスのシークレットを表示
aws secretsmanager list-secrets \
  --query "SecretList[?starts_with(Name, '${ENV}/wordpress/')]"
```

### 4.4 シークレットの命名規則
- 形式: `<環境>/<アプリケーション名>/<用途>-<ランダムサフィックス>`
- 例:
  - dev/wordpress/db-password-abc123
  - dev/wordpress/db-host-xyz789
- 注意点:
  - シークレット参照時はサフィックスを含めた完全なARNを使用
  - IAMポリシーではワイルドカードを使用可能

## 6. ロールバック手順

### 6.1 アプリケーションのロールバック
```bash
# 以前のタスク定義のリビジョンを指定してサービスを更新
aws ecs update-service \
  --cluster ${ENV}-escforgate-cluster \
  --service ${ENV}-escforgate-service \
  --task-definition ${ENV}-escforgate-task:${PREVIOUS_REVISION}
```

### 6.2 コンテナイメージのロールバック
```bash
# 以前のイメージタグを指定してタスク定義を更新
aws ecs register-task-definition \
  --family ${ENV}-escforgate-task \
  --container-definitions '[{"name":"app","image":"${AWS_ACCOUNT_ID}.dkr.ecr.ap-northeast-1.amazonaws.com/escforgate-app:${PREVIOUS_TAG}"}]'
```

## 7. トラブルシューティング

### 8.1 一般的な問題と対処方法
- コンテナの起動失敗
- ヘルスチェック失敗
- メモリ不足
- 接続エラー

### 8.2 VPCエンドポイント関連の問題
1. 接続タイムアウト
   ```bash
   # セキュリティグループの確認
   aws ec2 describe-security-groups \
     --group-ids ${SECURITY_GROUP_ID} \
     --query 'SecurityGroups[*].{Inbound:IpPermissions,Outbound:IpPermissionsEgress}'

   # VPCエンドポイントの状態確認
   aws ec2 describe-vpc-endpoints \
     --filters Name=vpc-id,Values=${VPC_ID}
   ```

2. 権限エラー
   ```bash
   # タスク実行ロールの確認
   aws iam get-role-policy \
     --role-name ${TASK_EXECUTION_ROLE} \
     --policy-name SecretsAccess

   # タスクロールの確認
   aws iam get-role-policy \
     --role-name ${TASK_ROLE} \
     --policy-name AppPolicy
   ```

3. DNS解決の問題
   ```bash
   # VPCのDNS設定確認
   aws ec2 describe-vpc-attribute \
     --vpc-id ${VPC_ID} \
     --attribute enableDnsHostnames

   aws ec2 describe-vpc-attribute \
     --vpc-id ${VPC_ID} \
     --attribute enableDnsSupport
   ```

### 8.2 ログの確認方法
```bash
# CloudWatchログの確認
aws logs get-log-events \
  --log-group-name /ecs/${ENV}-escforgate \
  --log-stream-name ecs/app/${TASK_ID}
```

### 8.3 ECSExecを使用したデバッグ
```bash
# コンテナへの直接アクセス
aws ecs execute-command \
  --cluster ${ENV}-escforgate-cluster \
  --task ${TASK_ID} \
  --container app \
  --command "/bin/bash" \
  --interactive
