## 2. 手動設定が必要なリソース

### **ドメインとSSL証明書**
- ドメイン取得（Route 53または他のレジストラ）
- ACM証明書のドメイン検証
- Route 53のDNSレコード設定

### **ACM証明書の事前準備**
- **目的**: CloudFormationデプロイ前にSSL証明書を準備し、HTTPS通信を可能にする
- **手順**:
  1. **証明書の作成**:
     ```bash
     # ACM証明書の作成
     aws acm request-certificate \
       --domain-name dev.tk-niigata.com \
       --validation-method DNS \
       --region ap-northeast-1
     ```
  2. **検証レコードの設定**:
     - ACMコンソールで提供されるCNAMEレコードを確認
     - DNSプロバイダーでCNAMEレコードを設定
     - 検証完了まで待機（通常5-30分）
  3. **証明書ARNの取得**:
     ```bash
     # 証明書一覧から対象ドメインの証明書ARNを取得
     aws acm list-certificates \
       --region ap-northeast-1 \
       --query 'CertificateSummaryList[?DomainName==`dev.tk-niigata.com`].CertificateArn' \
       --output text
     ```
  4. **CloudFormationテンプレートでの参照**:
     - compute.yamlのListenerHTTPSリソースで証明書ARNを参照
     ```yaml
     ListenerHTTPS:
       Type: AWS::ElasticLoadBalancingV2::Listener
       Properties:
         Certificates:
           - CertificateArn: !ImportValue dev-escforgate-certificate-arn
     ```

- **注意事項**:
  - 証明書は必ずCloudFormationデプロイ前に作成すること
  - 証明書の検証が完了していることを確認してからデプロイを開始
  - 証明書のARNはCloudFormationのパラメータまたはSecretsManagerで管理

### **ALBのSSL設定方針**
- **目的**: ALB（Application Load Balancer）を使用して、HTTPSを通じて安全にトラフィックを処理する
- **手順**:
  1. **ACMでの証明書取得**:
     - `dev.tk-niigata.com`のSSL証明書をAWS Certificate Manager（ACM）で取得
     - DNS検証を使用して証明書を検証
  2. **ALBのリスナー設定**:
     - ALBのリスナーにHTTPS設定を追加し、ACMで取得した証明書を使用
     - CloudFormationまたはTerraformを使用して、リスナー設定をコード化
  3. **セキュリティの強化**:
     - SSLポリシーを設定し、最新のセキュリティ基準に準拠

- **注意事項**:
  - 証明書のARNは、CloudFormationまたはTerraformのテンプレートで動的に取得
  - 証明書の自動更新をACMで設定し、手動での更新作業を省略

### **初期データとアカウント**
- 管理者アカウントの作成
- 初期データのセットアップ
- テストユーザーの作成

### **監視とアラート**
- CloudWatch アラーム
- SNSトピック
- メール通知設定

**注意事項**:
- サンプル段階では監視とアラートの設定をコード化する必要はありません
- 実際の構築が始まった際には、以下の項目をコード化することを推奨します：
  - CloudWatchアラームの設定
  - SNSトピックの作成と設定
  - メール通知の設定
- コード化により、全環境での一貫性を保ち、管理を容易にします

## 8. サンプル構築の方針

### **目的**
- サンプル構築では、最小限のリソースを使用して基本的な動作を確認します
- 監視やアラート等の運用機能は含めず、基本機能の動作検証に焦点を当てます

### **使用するリソース**
- **ECS**: 
  - コンテナ化されたアプリケーションのデプロイと管理
  - 基本的なオートスケーリング設定

- **RDS**: 
  - データベースの管理とデータの永続化
  - マルチAZ構成での基本的な冗長性確保

- **Cognito**: 
  - ユーザー認証と管理
  - 基本的な認証フローの確認

### **構築の範囲**
- 上記の最小限のリソースを使用して、基本的なWebサービスが動作することを確認
- 各コンポーネント間の連携が正常に機能することを検証
- 本番環境で必要となる高度な設定や運用機能は、サンプル構築後に段階的に追加
