# ECSForgate プロジェクト 用語集

## 1. プロジェクト固有の用語

| 用語 | 定義 |
|------|-----|
| ECSForgate | AWS ECS Fargateを利用したプライベート接続APIサービスを構築するプロジェクトの名称 |
| 確認記録 | 本APIサービスで管理する主要データエンティティ、特定の事象や状態の確認情報を記録するもの |
| JWT認証 | JSON Web Tokenを使用したAPI認証方式、トークンベースの認証を実現する |

## 2. AWS関連の用語

| 用語 | 定義 |
|------|-----|
| ECS (Elastic Container Service) | AWSのコンテナオーケストレーションサービス、Dockerコンテナを実行・管理する |
| Fargate | サーバーレスコンテナ実行環境、コンテナ実行に必要なインフラを管理不要にするECSの実行モード |
| RDS (Relational Database Service) | AWSのマネージドリレーショナルデータベースサービス、PostgreSQLなどの各種DBエンジンをサポート |
| CloudFormation | AWSリソースをコードとして定義・プロビジョニングするためのInfrastructure as Code (IaC)サービス |
| VPC (Virtual Private Cloud) | AWSクラウド内の仮想ネットワーク、独立したネットワーク空間を提供する |
| ALB (Application Load Balancer) | HTTPおよびHTTPSトラフィックを処理するロードバランサー、レイヤー7（アプリケーション層）で動作 |
| API Gateway | APIの作成、公開、維持、モニタリング、保護のためのフルマネージドサービス |
| CloudWatch | AWSリソースのモニタリング、ログ収集、メトリクス分析、アラート通知を提供するサービス |
| SNS (Simple Notification Service) | メッセージングとモバイル通知のためのフルマネージドのPub/Subメッセージングサービス |
| X-Ray | アプリケーションの分析とデバッグを支援する分散トレースシステム |
| ECR (Elastic Container Registry) | Dockerコンテナイメージを保存、管理、デプロイするためのフルマネージドコンテナレジストリ |
| IAM (Identity and Access Management) | AWSサービスとリソースへのアクセスを安全に制御するための認証・認可サービス |
| KMS (Key Management Service) | 暗号化キーの作成と管理を簡素化するセキュリティサービス |
| AZ (Availability Zone) | 独立した電源、ネットワーク、冷却機能を持つ、地理的に分離されたデータセンター |
| Auto Scaling | 需要に応じて自動的にコンピューティングリソースを調整する機能 |

## 3. アーキテクチャ関連の用語

| 用語 | 定義 |
|------|-----|
| マイクロサービス | 小さく独立したサービスの集合体としてアプリケーションを構築するアーキテクチャスタイル |
| RESTful API | REpresentational State Transferの原則に従ったAPIデザイン、リソース指向のインターフェース |
| IaC (Infrastructure as Code) | インフラストラクチャをコードとして管理・プロビジョニングする手法 |
| CI/CD | 継続的インテグレーション(CI)と継続的デリバリー(CD)、自動化されたビルド・テスト・デプロイプロセス |
| 水平スケーリング | インスタンス数を増やすことでシステム全体の処理能力を向上させる方法 |
| 垂直スケーリング | 既存インスタンスのリソース（CPU、メモリなど）を増強する方法 |
| マルチAZ | 複数のアベイラビリティゾーンにわたってシステムを分散させる高可用性構成 |

## 4. 開発・運用関連の用語

| 用語 | 定義 |
|------|-----|
| Spring Boot | Javaベースのアプリケーションフレームワーク、特にマイクロサービス開発に適している |
| Docker | コンテナ技術を使用してアプリケーションをパッケージ化・実行するプラットフォーム |
| GitHub | Gitリポジトリのホスティングサービス、バージョン管理やCI/CD連携が可能 |
| GitHub Actions | GitHubに統合されたCI/CDサービス、ワークフローの自動化を実現する |
| SLA (Service Level Agreement) | サービスの品質や可用性に関する合意事項、通常は稼働率などの数値目標を含む |
| RTO (Recovery Time Objective) | 障害発生後にシステムを復旧させるまでの目標時間 |
| RPO (Recovery Point Objective) | 障害発生時に許容できるデータ損失の最大期間 |
| ブルー/グリーンデプロイ | 新旧2つの環境を用意し、切り替えることでダウンタイムを最小化するデプロイ手法 |
| カオスエンジニアリング | 障害を意図的に発生させて、システムの耐障害性を検証するアプローチ |

## 5. セキュリティ関連の用語

| 用語 | 定義 |
|------|-----|
| FISC (金融情報システムセンター) | 金融機関のシステムに関するセキュリティガイドラインを策定する組織 |
| FISC安全対策基準 | 金融機関のコンピュータシステムのセキュリティ対策に関するガイドライン |
| TLS (Transport Layer Security) | データ通信の暗号化プロトコル、旧SSL |
| 最小権限の原則 | 必要最小限の権限のみを付与するセキュリティ原則 |
| SAST (Static Application Security Testing) | アプリケーションコードの静的セキュリティ解析 |
| DAST (Dynamic Application Security Testing) | 実行中のアプリケーションに対する動的セキュリティ解析 |

## 6. データベース関連の用語

| 用語 | 定義 |
|------|-----|
| PostgreSQL | オープンソースのオブジェクトリレーショナルデータベース管理システム |

| トランザクション | データベースの一貫性を保つための論理的な処理単位 |

## 7. モニタリング・運用関連の用語

| 用語 | 定義 |
|------|-----|
| メトリクス | システムのパフォーマンスや状態を示す計測可能な値 |
| アラート | 事前に定義された条件を満たした際に通知される警告 |
| ログ | システムやアプリケーションの動作記録 |
